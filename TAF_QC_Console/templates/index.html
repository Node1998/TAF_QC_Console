TAF format reference

AFMAN 15-124 (16 Jan 2019), Chapter 1 · unit TAF encoding guide. Findings cite the governing paragraph or table.

Message template

What the engine checks

  • Header — one AMD/COR modifier max (1.3.2.1.2), ICAO (1.3.2.1.3), issue time DDHHMMZ (1.3.2.1.4), valid period 24/30 h (1.3.2.1.5).
  • Wind — direction to 10°, calm = 00000KT, VRB only ≤6 KT (1.3.4.1), gust criteria (1.3.4.2.2).
  • Visibility — Table 1.1 reportable meters; w’w’ required when <9999 (1.3.5).
  • Weather — Table 1.2 column order, max three groups (1.3.6.2), VC pairing without intensity (1.3.6.4).
  • Sky — ascending bases, summation principle (1.3.7.1), Table 1.3 height increments, CB required with TS (1.3.7.8).
  • Altimeter — QNH…INS on INIT/FM/BECMG lines; never in TEMPO (1.3.12).
  • LLWS — WS group prohibited in BECMG/TEMPO (1.3.9.2.2).
  • Change lines — BECMG ≤2 h (1.3.3.1); TEMPO ≤6 h and ≤2 consecutive BECMG/TEMPO (encoding guide).
  • Temperature — TX before TN, M prefix below zero (1.3.13.1).

Scoring

Every TAF starts at 100. Each discrepancy subtracts its severity-weighted penalty (shown as −n beside the finding). Any critical finding or a score below 70 → FAIL; any error or a score below 90 → REVIEW; otherwise PASS. Highlighted tokens in the forecast show exactly where each penalty was taken.
